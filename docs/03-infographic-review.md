## ۱. شرح کلی

**IMAGE** یک وب‌اپلیکیشن استاتیک است که بدون نیاز به سرور اختصاصی، مستقیماً از مرورگر کاربر اجرا می‌شود. هدف اصلی، ایجاد تجربه‌ای تعاملی و سینمایی است: کاربر با یک کلیک، فرآیند ضبط تصویر را آغاز می‌کند؛ تصویر به‌صورت خودکار به **سرور ابری** ارسال می‌شود و سپس ویدیوی از پیش تعریف‌شده پخش می‌گردد.

این پروژه با رویکرد **Serverless Frontend** طراحی شده — یعنی منطق اصلی در سمت کلاینت (HTML، CSS، JavaScript) اجرا می‌شود و تنها برای ذخیره‌سازی و توزیع محتوا از زیرساخت ابری استفاده می‌کند. هاستینگ روی **GitHub Pages** با استقرار خودکار از طریق GitHub Actions انجام می‌شود.

## ۲. اهداف و کاربرد

| هدف | توضیح |
|-----|--------|
| **تعامل بصری** | ایجاد تجربه Play-to-Capture با رابط Frost Glass و افکت Film Grain |
| **ذخیره‌سازی ابری** | آپلود خودکار تصاویر به سرور ابری بدون واسطه backend |
| **جمع‌آوری متادیتا** | ثبت نوع دستگاه، زمان capture و اطلاعات مرورگر |
| **پخش محتوا** | نمایش ویدیو از CDN سرور ابری پس از اتمام فرآیند |
| **استقرار ساده** | بدون npm، بدون build، deploy مستقیم فایل‌های خام |

## ۳. معماری فنی (وضعیت فعلی)

```
کاربر → GitHub Pages (HTTPS)
              │
      ┌───────┴───────┐
      ▼               ▼
 getUserMedia    سرور ابری (Upload API)
 (دوربین)              │
      │           فولدر IMAGE
 Canvas → JPEG    + tags + context
                          │
                    playbackVideoUrl (CDN)
```

**فایل‌های اصلی:** `index.html` (UI/RTL)، `app.js` (منطق capture و آپلود)، `config.js` (پیکربندی سرور ابری)

**مشخصات capture:** JPEG با کیفیت ۹۲٪، رزولوشن ایده‌آل 1920×1080، تأخیر ۲ ثانیه قبل از ثبت فریم، capture بدون نمایش تصویر زنده به کاربر.

---

## ۴. امنیت

- آپلود از طریق **Unsigned Preset** محدودشده (folder، format، size)
- عدم استفاده از API Secret در فرانت‌اند
- ارتباط صرفاً روی HTTPS
- متادیتا در context/tags سرور ابری — نه در نام فایل

## ۵. اهداف آینده

| فاز | هدف | توضیح |
|-----|------|--------|
| **فاز ۱** | پنل کنترل (Admin Dashboard) | مشاهده، مدیریت و دسته‌بندی تصاویر آپلودشده؛ احراز هویت admin |
| **فاز ۲** | مدیریت لینک و آمار ترافیک | تولید لینک اختصاصی برای هر کمپین؛ ثبت بازدید، منبع ورود و نوع دستگاه |
| **فاز ۳** | Backend میانی | سرویس Node.js یا Supabase/Firebase برای metadata، IP geolocation (با consent)، و webhook از سرور ابری |
| **فاز ۴** | گسترش permissionها | دسترسی میکروفون (ضبط صدا)، موقعیت مکانی (Geolocation API)، اعلان push — هر کدام با prompt صریح مرورگر |
| **فاز ۵** | نسخه موبایل (PWA / Native) | دسترسی به قابلیت‌های native در صورت نیاز به features خارج از محدودیت Web API |

## ۶. محدودیت‌های فعلی

- وابستگی به HTTPS و مجوز دوربین
- عدم وجود backend برای analytics پیشرفته
